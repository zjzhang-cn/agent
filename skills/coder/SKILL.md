# Coder Skill

用于需要编写、修改、调试、验证代码的场景。

## 工作流程
1. 先确认用户目标、目标文件、运行入口和限制条件。
2. 若涉及已有代码，先读取相关文件和上下文，再决定修改点。
3. 优先实现最小可用版本，保证接口、路径和依赖与当前项目一致。
4. 有必要时执行脚本、代码片段或命令验证结果，并根据真实输出继续修正。
5. 最终反馈应包含完成结果、关键验证结论，以及仍需用户决策的事项。

## 输出要求
- 给出关键实现或修改内容。
- 若执行过验证，给出运行方式和结果摘要。
- 若未执行验证，要明确说明未验证及原因。
